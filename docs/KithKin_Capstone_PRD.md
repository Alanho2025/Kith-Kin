# Kith&kin (KK) — 最终版 PRD / 团队对齐文档

> 比赛：Google × Kaggle《AI Agents: Intensive Vibe Coding》Capstone
> **赛道（已锁）：Concierge Agents**
> **目标：获奖。一切取舍只服务"拿奖"，不考虑商业化。**
> 一句话定位：**当子女在上班、父母独自在外时，替你在场——陪伴随迁/探亲的年迈父母，走过在异国"听不懂、说不出"的高风险时刻。**
> 架构：**单条 Gemini Live API 会话（Agent 模式）+ ADK 多智能体编排 + 文字翻译旁路（视听分轨）。**

## 0. 为什么是这个方案（评分逻辑，先讲透）

评分细则三条硬约束决定一切：

1. **70 / 100 分在实现（架构 + 代码 + 工具调用），只有 30 分在故事。** → 必须交工程扎实、真在编排工具的 agent。
2. **必须命中"至少 3 个课程概念"。** → 锁 **ADK 多智能体 + MCP Server + Security features**，恰好是我们已有的强项（Hermes Agent、MCP、SOUL.md 注入防御）。
3. **最高权重判分语：「agent 的使用必须清晰、有意义、且处于核心」。** → 场景必须是"翻译机做不到、非 agent 不可"的场景。**这条决定了我们选药房。**

## 2. 产品定义（MVP Scope）

KK 是一个**单管道、视听分轨的陪伴 agent**：

- **看（视觉听轨 / 纯忠实）**：屏幕实时滚动**超大字号中文**，是对方原话的**忠实直译**，不经 KK 加工。
- **听+卡（语音答轨 / KK 军师）**：KK 用温和中文语音提示风险，并下发 2–3 张**可确认**回应卡（附回译）；父母点选确认后由 TTS 代说，也可随时按"我自己说"。
- **家属在环**：就诊结束后，KK 把结构化结果写回记忆，并给子女发摘要；下次主动提醒。

**旗舰 demo 场景：父母独自去社区药房，复购降压药 / 问一款非处方药。**

## 3. 卖点、情感内核与英雄镜头（写进 Writeup 和视频）

**四个卖点**
1. **视听分轨 UX**——眼睛看忠实翻译（保真、零幻觉），耳朵+卡片交给有温度的 KK（加工、给选项）。一条管线实现"安全"与"温度"不冲突。
2. **真·跨会话记忆**——通过 MCP `memory` 工具读写，agent *自己决定*何时检索、何时写回；不是写死的 JSON。
3. **家属在环 + 隐私优先**——健康/财务/紧急数据"播报需确认"，永不擅自替父母编话；正中 Concierge "safe and secure"。
4. **Agent-native**——单 Live API 会话 + ADK 多智能体 + MCP 工具编排，不是翻译软件套壳。

**情感内核（视频灵魂，一句话）**
> 每个移民子女都怕那一刻：父母独自站在柜台前，听不懂、说不出，而你在上班。**KK 就是那个替你在场的人。**

**英雄镜头（demo 高潮）**
> 第一次：父母复购降压药，药剂师提到一款新药。KK 调记忆+查药典发现可能与现有用药冲突 → 屏幕大字忠实显示药剂师原话 → KK 语音提示"这药可能和您的降压药冲突" → 推卡"请帮我确认是否冲突" → 事后把"药剂师建议+新药名"写回记忆并发给女儿。
> **第二次：父母再来，KK 主动开口："上次那款药要先确认冲突——今天要不要一起问清楚？"**
> 一镜证明：① 真 agent（自主检索/写回）② 真记忆（跨会话）③ Concierge 价值（管药+家属在环）④ 情感暴击。

## 4. 技术架构（50 分主战场：单流基底 + 多智能体编排 + 视听分轨）

**核心认知（架构融合）**：摒弃高风险的"双独立并发音频流"，全面切换至 **Gemini Multimodal Live API（带 Function Calling）** 建立**唯一一条 WebSocket 音频连接**。官方底座解决网络/通信；在数据消费层做"视听分轨"。

### 4.1 视听分轨机制（单管道上的双轨 UX）⭐含关键补丁

一条 WebSocket 音频会话，两种数据消费，彻底解决"安全保真"与"温度加工"的矛盾：

- **视觉听轨（纯忠实，无幻觉）**：Live API 原生吐出 `input_transcription`，**注意——这是对方说话的英文原文转录，不是中文**。要得到屏幕上的忠实中文，必须挂一个**轻量"文字→文字"翻译旁路**（Flash 级 text 调用即可），把英文转录实时译成中文上屏。
  - **这是文字侧信道，不是第二条音频流**：无音频时间同步问题、成本极低。
  - **红线**：忠实翻译由这条解耦旁路产出，**绝不让 KK agent 顺手输出**——否则加工/幻觉风险回流。
- **语音/卡片答轨（KK 军师，主控全场）**：仅当检测到事件（风险、问询、停顿）时，ADK 编排引擎才发号施令——TTS 播报警告，或通过函数调用/JSON 下发渲染 2–3 张可确认卡。

> ⚠️ D1 必验：在 AI Studio 确认 agent 模式下 `input/output transcription` 的实际内容与可截取性，确认文字翻译旁路链路通。这是整套架构唯一的技术不确定点。

### 4.2 ADK 多智能体编排（Kaggle 核心得分点）——必交概念 ①

不写单体大 Prompt，**显式定义多智能体**以锁定课程概念分：

- **Orchestrator（主路由）**：监听转录文本流，判断"顺畅翻译中"还是"需要决策"，拉起下游 Agent。
- **Companion（KK 核心）**：理解上下文，调用工具组装可确认卡片。**绝对红线：只做风险提示与选项生成，绝不代替药剂师转述病情事实。**
- **Guardian（安全与门控）**：独立一层。
  - *英雄拦截镜头*：以**意图分类为主、正则为兜底**（不要纯关键词硬编码），一旦药剂师询问"信用卡、护照号、家庭住址"，切断普通回路，强制推卡 `[ 涉及隐私，KK 已拦截，按此拒绝 ]`。
  - 向子女发信息的二次确认门控。

> 技术栈：**Gemini + Google ADK**（ADK 走 Streaming 接 Live API）。**务必确认是真的在用 ADK**，否则该概念不算数。

### 4.3 MCP 工具层 + 极限时延优化——必交概念 ②

面对面交流对延迟零容忍，Tool Use 必须预加载或异步隐蔽：

- MVP MCP 只暴露四个工具：`memory_search`、`memory_write`、`check_drug_interaction`、`notify_family`。
- **预热记忆（Pre-fetch）**：会话一建立，后台静默执行 `memory_search(query="profile", tags=["profile", "medications", "allergies"])`，并按 RAG 上限只把当前回合需要的片段放入上下文。
- **条件触发（Lazy-load）**：`check_drug_interaction` 仅在转录流捕获到具体药品实体后异步触发，用极短 TTS（"KK 正在帮您核对…"）掩盖请求延迟。药名缺失时必须请药剂师确认或写下药名，不调用额外知识工具，也不猜测。

### 4.4 Security features——必交概念 ③（同时是 Concierge 核心价值）

- **App WebSocket ticket（应用票据）**：MVP 浏览器只连接 FastAPI，并使用后端签发的短效、单次、session-bound ticket；Gemini API key 只在后端。Gemini ephemeral token 仅属于未来 client-direct 拓扑，不在 MVP 宣称支持。
- **Prompt-injection 防御（SOUL.md 式）**：对方话术或注入文本无法让 KK 泄露父母数据或给出有害建议。
- **隐私分级 + 同意门控 + 零 PII 日志**：健康/财务数据播报前必须确认+回译；通知家属需二次确认；日志/URL 不带个人数据。

> **可选第 4 概念（低成本加分）**：用 "Gemini Live API Dev" skill 开发可命中 **Agent skills**；部署上线再命中 **Deployability**。

## 5. 设计红线（既是对的，也是 Concierge 加分项）

1. **KK 绝不替父母说他没确认过的话，也绝不转述药剂师的事实陈述。** 事实信息走屏幕忠实大字；KK 只做"风险提示 + 给选项卡"。
2. **父母随时可"我自己说"。** 自主权高于一切。
3. **不给医疗"该怎么办"的判断。** 范围锁死，避开责任地雷。
4. **健康/财务数据本地/临时优先，最小披露。**

## 6. 明确砍掉的东西（保 MVP 不翻车）

| 砍 | 原因 | 替换方案 |
|---|---|---|
| **Gradio / Streamlit 前端** | 扛不住全双工低延迟 PCM 实时推拉流，会卡顿+状态机死锁 | **官方 WebSocket 示例**（JS 前端 + Python 后端 + 临时令牌）为唯一底座 |
| **双独立 WebSocket 缝合架构** | 两流并发→时间戳错位、上下文撕裂、Token 成本翻倍 | 融合为**单一 Live API 会话**，前端做"视听分轨"渲染 |
| **自建全量状态机（VAD/轮次接管）** | 14 天手写完美停顿检测极易抢话/死机 | 用 Live API **原生端点检测** + UI"手动打断/确认"按钮兜底 |
| **KK 扮"口技演员"（转述原话）** | 医疗/法律场景致命幻觉 | 事实走屏幕大字直给；KK 只提示风险+给卡 |
| **方言 / 自建 ASR / 多模态超市识别** | 增工程、不增分 | 普通话；忠实翻译走文字旁路；超市仅 15 秒热身 |

## 7. 评委视角的扣分项与防守（开会重点过这一节）

- **7-1 场景错配是最隐蔽的自残**：完美问路翻译器 < 略糙药房 agent，因为前者过不了"为什么需要 agent"门槛。药房主演，问路只当热身。
- **7-2 头号扣分杠杆"是不是真 agent"**：Router/Companion/Guardian 必须用 ADK 真编排，让 agent 自主选工具、多步推理；删掉一切关键词硬编码。
- **7-3 Concierge"safe and secure"是定向考点**：评委会特意找隐私机制。把 SOUL.md、临时令牌、播报前确认、信用卡熔断摆到台面上讲。
- **7-4 没有评估 = 安静扣分**：放十几条 eval set + 几张决策 trace 截图。
- **7-5 叙事漏洞**：视频必须交代"子女上班、父母独自"。

## 9. 14 天日程（草案，按人手调整）

- **D1–2｜对齐 + 验证命门**：起 ADK 项目；克隆官方 WebSocket 示例；**立刻在 AI Studio 验证 agent 模式 transcription 内容 + 文字翻译旁路链路 + 中英质量 + 配额**。
- **D3–4｜单流基底**：Live API 单会话收发音频 + 原生端点检测 + 大字 UI；文字旁路把英文转录译成中文上屏（视觉听轨通）。
- **D5–7｜ADK + MCP 核心**：Orchestrator+Companion；MCP 的 memory/knowledge/check_interaction 接通；可确认卡 + 回译出得来；预热/懒加载延迟优化。
- **D8–9｜Guardian + 安全**：意图分类拦截、信用卡熔断、临时令牌、同意门控、注入防御。
- **D10｜英雄镜头联调**：写回→下次回忆闭环跑顺。**兜底检查点：没通就降级（见 §10）。**
- **D11–12｜评估 + 文档**：eval set、trace 截图、README、架构图、代码注释。
- **D13｜录 demo + 剪视频**（≤5 分钟，发 YouTube）。
- **D14｜Writeup + 挂全部资产 + 提交**（务必点 Submit，草稿不算；留缓冲）。
